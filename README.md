# AR Radius

A modern, lightweight FreeRADIUS management console for Ubuntu 24.04. Built with **PHP + MariaDB + FreeRADIUS** and a dark, mobile-friendly UI. Designed to be simple and focused — no DaloRADIUS feature bloat.

---

## Features

- **User Management** — Add / edit / delete RADIUS users (username, password, optional expiry).
- **Simultaneous Use** — Set `Simultaneous-Use := N` directly to the `radcheck` table.
- **User Status** — Online / offline indicator, open session count, one-click **Disconnect** button (RADIUS Disconnect-Message via `radclient`).
- **Dashboard** — Total users, online users, active sessions, daily auth stats.
- **Self-Update** — "Check for Updates" button pulls latest code from GitHub and runs `update.sh`.
- **One-Command Installer** — Fresh install on Ubuntu 24.04 with a single `curl | bash`.
- **REST API** — JSON endpoints under `/api/` for users, sessions, disconnects, stats.
- **Hardened** — CSRF protection, bcrypt admin passwords, session timeout, audit log, UFW firewall.

## Quick Install

On a fresh Ubuntu 24.04 server, as root:

```bash
curl -sSL https://raw.githubusercontent.com/marsh4200/ar-radius/main/install.sh | sudo bash
```

The installer will:

1. Install **FreeRADIUS**, **MariaDB**, **Apache**, **PHP** and required modules.
2. Create the `radius` database and import the schema (FreeRADIUS tables + AR Radius tables).
3. Configure the FreeRADIUS **SQL module** to use MariaDB.
4. Configure **Apache** with a vhost on **port 9916**.
5. Open ports `9916/tcp`, `1812/udp`, `1813/udp`, `22/tcp` in **UFW**.
6. Enable and start all services.
7. Create an **admin** account with a strong random password.
8. Save credentials to `/root/ar-radius-credentials.txt`.

When it finishes you'll see something like:

```
=========================================================
  AR Radius installation complete!
=========================================================

  Web GUI : http://<server-ip>:9916/
  Login   : admin
  Pass    : <random-strong-password>

  Credentials saved to: /root/ar-radius-credentials.txt
=========================================================
```

## REST API

All endpoints require an authenticated session cookie. Mutating requests require the `X-CSRF-Token` header (token is in the `<meta name="csrf-token">` tag on every page).

| Method | Path | Purpose |
| ------ | ---- | ------- |
| `GET`    | `/api/stats.php` | Dashboard counters |
| `GET`    | `/api/users.php` | List all users |
| `GET`    | `/api/users.php?username=foo` | Get one user |
| `POST`   | `/api/users.php` | Create user |
| `PUT`    | `/api/users.php?username=foo` | Update user |
| `DELETE` | `/api/users.php?username=foo` | Delete user |
| `POST`   | `/api/disconnect.php` | Disconnect a session |
| `GET`    | `/api/update.php?action=check` | Check for updates |
| `POST`   | `/api/update.php?action=run` | Run `update.sh` |

User create example:
```bash
curl -X POST http://server:9916/api/users.php \
    -H 'Content-Type: application/json' \
    -H "X-CSRF-Token: $CSRF" \
    -b cookies.txt \
    -d '{"username":"alice","password":"s3cret","simul_use":1,"expiry":"2026-12-31T23:59"}'
```

## Update System

Click **System → Run Update** in the web UI, or run on the server:

```bash
sudo /opt/ar-radius/update.sh
```

The script:
1. `git fetch && git reset --hard origin/main`
2. Restores your local `web/includes/config.php` (kept out of git).
3. Re-applies the schema (idempotent — uses `CREATE TABLE IF NOT EXISTS`).
4. Fixes permissions.
5. Reloads Apache.

## GitHub Deployment Instructions

The single-command installer expects the repo to live at  
`https://github.com/marsh4200/ar-radius` on branch `main`.  
Here's how to publish it:

### One-time setup on your laptop

```bash
# 1. Create empty repo on GitHub (web UI):
#    Owner: marsh4200
#    Name : ar-radius
#    Visibility: Public  (must be public for raw.githubusercontent.com)
#    Do NOT add README/.gitignore/license from the GitHub UI - we have our own.

# 2. Clone this generated tree, then push it.
cd /path/to/ar-radius            # the folder with install.sh, web/, sql/, ...
git init -b main
git add .
git commit -m "Initial AR Radius release"
git remote add origin https://github.com/marsh4200/ar-radius.git
git push -u origin main
```

That's it. The installer URL is now live:

```
https://raw.githubusercontent.com/marsh4200/ar-radius/main/install.sh
```

### Publishing updates

Bump `VERSION`, commit, push:

```bash
echo "1.0.1" > VERSION
git add -A
git commit -m "Release 1.0.1: <what changed>"
git push
```

End-users hit **System → Check for Updates → Run Update** to pull it in.

> ⚠️ **Never commit `web/includes/config.php`** — it contains database credentials. It's already in `.gitignore` and is generated fresh by the installer on each host.

## NAS / Client Configuration

Add network devices either:

- **Via the `nas` table** (read at FreeRADIUS startup because `read_clients = yes` is set):
  ```sql
  INSERT INTO nas (nasname, shortname, type, secret, description)
  VALUES ('192.168.1.1', 'router1', 'other', 'sharedsecret123', 'Office router');
  ```
  Then restart FreeRADIUS: `sudo systemctl restart freeradius`.

- **Or via `/etc/freeradius/3.0/clients.conf`** the traditional way.

The "Disconnect" button uses the `secret` from the `nas` row to send Disconnect-Messages on UDP/3799.

## Security Notes

- The web GUI listens on **port 9916/tcp** only — restrict via firewall to your management network.
- The installer generates random passwords for both `admin` and the `radius` DB user.
- Admin passwords are stored bcrypt-hashed in `ar_admins.password_hash`.
- CSRF tokens are required on all mutating requests.
- An `ar_audit_log` table records every login, user create/edit/delete, disconnect, and update.
- The `www-data` sudoers rule is **narrow** — only `update.sh` and two `systemctl restart` commands.
- Consider putting Apache behind a reverse-proxy with TLS (Caddy / nginx) for production.

## Uninstall

```bash
sudo systemctl stop apache2 freeradius
sudo a2dissite ar-radius.conf
sudo rm -f /etc/apache2/sites-available/ar-radius.conf
sudo rm -f /etc/sudoers.d/ar-radius
sudo rm -rf /opt/ar-radius
sudo mysql -e "DROP DATABASE radius; DROP USER 'radius'@'localhost';"
sudo ufw delete allow 9916/tcp
sudo systemctl restart apache2
```
